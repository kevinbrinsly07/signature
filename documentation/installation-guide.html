SignMe Installation Guide

Version 1.1

System Requirements

  • Modern web browser with HTML5 Canvas support (Chrome 60+, Firefox 55+, Safari 11+, Edge 79+)
  • Web server (Apache, Nginx, or any hosting service)
  • Internet connection for CDN resources

Installation Steps

Step 1: Upload Files

Upload all the files from the SignMe package to your web server. You can place them in the root directory or a subdirectory.

Example directory structure after upload:

your-website.com/
├── index.html
├── stamp-designer.html
├── README.md
├── CHANGELOG.md
├── LICENSE.txt
├── js/
├── assets/
└── documentation/

Step 2: Set Permissions

Ensure your web server has read permissions for all files. For most hosting providers, this is set by default.

Step 3: Access the Application

Open your web browser and navigate to the URL where you uploaded the files.

Example: https://your-domain.com/signme/

The main page (index.html) will load the Document Signing tool by default.

Configuration

SignMe works out of the box without additional configuration. All resources are loaded from CDNs for optimal performance.

Optional: Local CDN Setup

If you prefer to host the CDN resources locally:

  1. Download Tailwind CSS, Font Awesome, PDF.js, and PDF-lib
  2. Update the CDN links in both HTML files to point to your local copies
  3. Update the asset paths if necessary

Troubleshooting

Canvas Not Working

Ensure your browser supports HTML5 Canvas. Update to the latest version of your browser.

Assets Not Loading

Check that all files were uploaded correctly and that file permissions are set properly.

CDN Resources Blocked

If CDN resources are blocked by your network, consider hosting them locally or using a different CDN.
